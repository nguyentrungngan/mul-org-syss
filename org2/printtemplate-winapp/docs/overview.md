# System Overview  

## Name  
**PrintTemplate**  

## Purpose  
PrintTemplate is designed to **standardize and manage sample forms** for payment requests, expense requests, and purchase requisitions.  
Each template is **assigned a unique sequential code**, cannot be deleted once created, and must be **approved before printing**. The system also provides **weekly reports** for audit tracking.  

## Key Features  
- **Unique Template Numbering**: Each template is assigned a unique sequential ID.  
- **Non-Deletable Records**: Once created, templates cannot be removed.  
- **Editable Before Approval**: Templates can be modified before approval.  
- **Restricted Editing After Approval**: Approved templates can only be modified if the approval is revoked by a manager.  
- **Approval-Based Printing**: Print function is enabled only after approval.  
- **Weekly Reports**: Generate reports on created and approved templates.  

## Target Users  
- **Department Staff**: Create, edit, and submit templates for approval.  
- **Managers**: Approve templates and revoke approval when changes are needed.  
- **Accountants**: Verify templates and track weekly reports for audits.
