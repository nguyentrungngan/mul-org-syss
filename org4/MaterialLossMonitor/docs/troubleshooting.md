# Troubleshooting Guide

## Purpose
This document provides troubleshooting steps for issues that may arise during the operation of the Modular Python Backend system. It covers common problems and their resolutions to help maintain system stability.

## Common Issues & Solutions

### 1. **Database Connection Issues**
   - **Problem**: The system fails to connect to the MES database.
   - **Cause**: Incorrect database credentials or network connectivity issues.
   - **Solution**:
     - Verify the database credentials in the `.env` file (ensure `MES_SERVER`, `MES_DB`, `MES_USER`, and `MES_PASS` are correct).
     - Check if the database server is reachable from the backend system.
     - Test the database connection using a database client like SQL Server Management Studio or DBeaver.
     - Ensure that firewalls or network settings are not blocking access.

### 2. **FTP Connection Problems**
   - **Problem**: Unable to connect to the FTP server or upload data.
   - **Cause**: Incorrect FTP server details or authentication failure.
   - **Solution**:
     - Double-check the `FTP_SERVER`, `FTP_USER`, and `FTP_PASS` variables in the `.env` file.
     - Ensure that the FTP server is up and running.
     - Verify that the correct path is set in `SFTP_REMOTE_PATH`.
     - Test the connection manually using an FTP client like FileZilla to ensure the credentials and paths are correct.

### 3. **Power BI Integration Failures**
   - **Problem**: Power BI dashboard fails to refresh or does not update with new data.
   - **Cause**: Incorrect Power BI credentials or API token.
   - **Solution**:
     - Ensure the `POWERBI_GROUP`, `POWERBI_DATASET`, and `POWERBI_ACCESS_TOKEN` are correctly set in the `.env` file.
     - Confirm that the Power BI API access token is still valid (you may need to regenerate the token if expired).
     - Test the API connection using Postman or any REST client to verify Power BI access.

### 4. **Email Notification Failures**
   - **Problem**: Email notifications are not sent (e.g., for alerts or reports).
   - **Cause**: SMTP server issues or incorrect email configuration.
   - **Solution**:
     - Verify the `EMAIL_SMTP`, `EMAIL_USER`, and `EMAIL_PASS` are correct in the `.env` file.
     - Check if the SMTP server is working properly (ensure the correct SMTP port and credentials).
     - Test sending an email from the configured SMTP server using a tool like Telnet or an email client.
     - Ensure the server allows sending emails from your specified email account.

### 5. **Data Synchronization Failures**
   - **Problem**: Data from PLCs, MES, or FTP servers is not being synchronized correctly.
   - **Cause**: Network issues, incorrect credentials, or configuration errors in the script.
   - **Solution**:
     - Ensure the `PLCS`, `MES_SERVER`, and `FTP_SERVER` environment variables are configured correctly.
     - Check if the Modbus or SQL Server connection is successfully established.
     - Review the logs for any errors related to data fetching and processing (check `system_logs.py` for detailed logs).

### 6. **VPN Connection Issues**
   - **Problem**: VPN connection command fails or does not establish the connection.
   - **Cause**: Incorrect VPN configuration or access issues.
   - **Solution**:
     - Ensure the `VPN_CONNECT_COMMAND` and `VPN_DISCONNECT_COMMAND` are correct in the `.env` file.
     - Test the VPN connection manually using the same commands in a terminal.
     - Verify that the VPN server is up and the credentials are valid.

## System Logs
The system logs errors and activities for debugging. Review the logs generated by `system_logs.py` to identify the root cause of issues. Logs will be located in the specified log folder or directory defined in the configuration.

- **Log Location**: Check the `log_directory` variable in the configuration file or `system_logs.py` to locate log files.
- **Error Messages**: Logs provide detailed error messages that will help identify specific issues.
